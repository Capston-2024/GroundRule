### 🌲 git 협업 전략
* 브랜치 관리: main > develop > feat/{개발 사항}
* feat/{개발 사항} 브랜치 작업 사항 merge 시, 반드시 PR 활용 (1명 이상의 approve 필수)
* develop 브랜치 작업 사항 merge 시, 반드시 PR 활용 (모두의 approve 필수)
* review 요청 시, 24시간 내에 응답
* PR 생성자가 merge & PR close
